## Spacing Scale

Declared values (must be multiples of 4) — confirms the existing Tailwind v4 default scale already used throughout the codebase (`px-4`, `px-6`, `py-3`, `gap-1.5`, etc.):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (tab-group padding, badge gaps) |
| md | 16px | Default element spacing (page/header horizontal padding) |
| lg | 24px | Section padding (`sm:px-6`) |
| xl | 32px | Layout gaps, criticality-picker touch target (`h-8 w-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (`py-8`/`py-12` container padding) |

Exceptions:
- Checkbox touch target is `20px` (`h-5 w-5`, `check-box.tsx`) — pre-existing component dimension, off the 8pt scale but **not touched by this phase** (Phase 1 only re-tokenizes color/motion on `criticality-picker.tsx`/`nav-tabs.tsx`/`view-tabs.tsx`; component sizing is unchanged everywhere per the functional-parity constraint).
- Criticality-picker's dot button is `32px` (`h-8 w-8`) — on-scale, but below the 44px touch-target guideline. Pre-existing, unresized this phase (FOUND-03 only recolors the dot, not its hit area).

## Typography

Root `font-size` scales from `100%` (16px) to `112.5%` (18px) at `≥1024px` (FOUND-04, lever 1 — see 01-RESEARCH.md "Desktop type-scale + density"). All sizes below are declared as the mobile-base rem value; the desktop column shows the computed px after the `lg:` root-scale multiplier (×1.125) — no component-level `lg:text-*` overrides are introduced this phase (D-07).

| Role | Size (rem / mobile px) | Desktop px (≥1024px) | Weight | Line Height |
|------|------|--------|--------|-------------|
| Label | 0.875rem / 14px (`text-sm`) | 15.75px | 500 (medium) | 1.4 |
| Body | 1rem / 16px (`text-base`) | 18px | 500 (medium) | 1.5 |
| Heading | 1.125rem / 18px (`text-lg`) | 20.25px | 600 (semibold) | 1.2 |
| Display | 1.875rem / 30px (`text-3xl`) | 33.75px | 600 (semibold) | 1.2 |

- Weights: exactly 2 — `500` (medium, default UI/body text — matches existing `font-medium` usage in `nav-tabs.tsx`/`view-tabs.tsx`) and `600` (semibold, headings/emphasis/stat numbers — matches existing `font-semibold` usage). No `400`/`700` introduced.
- Micro/meta text (`text-xs`, 12px) already exists in the codebase (timestamps, badges) and is unchanged — it sits below the `Label` role and is not re-scaled by the desktop lever since it's reserved for secondary/de-emphasized text, not primary reading content.
- Display role (stat numbers, e.g. Statistics page counts) gets `tabular-nums` treatment per BRAND-02 in a later phase — Phase 1 only establishes the size/weight/line-height token, not the count-up animation (that's Phase 3 scope).

## Color

Two accent mechanisms are locked by CONTEXT.md (D-01/D-02) and must stay visually distinct — do not blend them:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background: oklch(0.16 0.006 75)` (warm-neutral tier 0) | Page background |
| Secondary (30%) | `--card: oklch(0.22 0.008 75)`, `--surface: oklch(0.19 0.007 75)`, `--surface-2: oklch(0.25 0.009 75)`, `--popover: oklch(0.29 0.010 75)` (tiers 1-3) | Cards, header/nav surface, raised modals, popovers/dropdowns/tooltips — 4 distinct elevation lightness steps per FOUND-02 |
| Accent — Action color (10%) | `--primary: oklch(0.72 0.17 48)` (warm amber/orange, unchanged per D-01) | Primary CTA buttons, focus rings (`--ring`), active checkbox fill, active nav-tab underline. Never used for criticality/category coding. |
| Accent — Brand gradient (functional, part of the same 10%) | `--gradient-teal: oklch(0.74 0.11 195)` → `--gradient-violet: oklch(0.66 0.19 322)` (`--gradient-brand` linear-gradient) | **Reserved exclusively for:** criticality-picker dots/borders (`on_track`=teal end, `warning`=midpoint `oklch(0.70 0.14 258)`, `critical`=violet end), the single-corner ambient background glow (FOUND-05), and active-state borders/highlights on selected criticality options. Never applied to primary buttons or generic decorative surfaces (D-02). |
| Destructive | `--destructive: oklch(0.62 0.2 25)` (unchanged) | Destructive actions only (delete task/habit/project) |

Accent reserved for: primary-action buttons + focus rings + active nav underline (amber `--primary`) — **and separately** — criticality/category/priority dots, ambient corner glow, active criticality-option border (teal→violet gradient tokens). These are the only two families of accent usage; no other element may introduce a third accent color this phase.

Contrast note (carried from 01-RESEARCH.md Assumptions A3): before this contract is implemented, re-verify `--foreground`/`--muted-foreground` against the new `--background`/`--card`/`--surface-2` values, and specifically the `warning` gradient midpoint (`oklch(0.70 0.14 258)`, a blue-family hue) against `--card`/`--surface-2` — flagged as the highest-risk token for failing 4.5:1/3:1 contrast.

## Copywriting Contract

Phase 1 (FOUND-01..06, MOTION-02/03) is a pure token/CSS/motion foundation phase — it introduces **zero new user-facing copy**. No new CTAs, empty states, error states, or destructive-action dialogs are created or reworded. Per PARITY-01, all existing copy across Tasks/Habits/Projects/Statistics must render byte-identical to today; only presentation (color/spacing/motion) changes.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — no new CTA introduced this phase; existing button labels (e.g. task/habit "Add", "Save") are unchanged |
| Empty state heading | Not applicable — no empty states introduced or modified this phase |
| Empty state body | Not applicable |
| Error state | Not applicable — no new error states this phase |
| Destructive confirmation | Not applicable — no destructive actions introduced this phase; existing delete/archive confirmations (if any) keep their current copy verbatim |
