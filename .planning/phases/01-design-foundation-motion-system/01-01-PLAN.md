<objective>
Establish the entire design-token substrate for the redesign in `globals.css`, plus a shared `motion.ts` constants file. This is the load-bearing foundation every downstream plan and phase consumes.

Three things happen here, in strict order, because each blocks the next:
1. **FOUND-01 cascade fix** — delete the two dead shadcn-default grayscale blocks so `:root` becomes the single source of truth. Until this deletion lands, every new `:root` token is silently overridden by `.dark` (which wins on `<html class="dark">` via source order).
2. **FOUND-02/03 palette** — re-tune `:root` for a warmer neutral base, 4 distinct elevation tiers, and add the teal→violet functional gradient tokens.
3. **FOUND-04/05/06 mechanics** — desktop root-font scaling, `--container-app` width token, motion duration/easing tokens, the global `prefers-reduced-motion` rule, the `.ambient-glow` CSS rule, and `src/lib/motion.ts`.

Purpose: Instantly change how the whole app reads (warmer, darker, layered, denser on desktop) and give components a token vocabulary to consume.
Output: A rewritten `globals.css` token layer and a new `src/lib/motion.ts`.
</objective>

    In the `@theme inline` block, add motion + container tokens following the existing `--radius-*` registration pattern: `--duration-micro: 150ms;`, `--duration-base: 200ms;`, `--duration-slow: 300ms;`, `--ease-out-soft: cubic-bezier(0.16, 1, 0.3, 1);`, and `--container-app: 88rem;`. These are documented Tailwind v4 theme namespaces that auto-generate `duration-micro`/`duration-base`/`duration-slow`/`ease-out-soft`/`max-w-app` utilities (same mechanism that already produces `max-w-5xl` from `--container-5xl`).

    Desktop type-scale + density (FOUND-04 lever 1, D-06): in the existing `@layer base` `html { @apply font-sans; }` rule, add `font-size: 100%;`. Then add a plain (non-@apply) media query: `@media (min-width: 1024px) { html { font-size: 112.5%; } }`. This scales ALL rem-based utilities (type + rem spacing) at ≥1024px — the "denser, fuller" desktop rhythm, not font-size alone.

    Global reduced-motion fallback (FOUND-06, the one genuinely new mechanism): append the canonical W3C C39 rule — `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; scroll-behavior: auto !important; } }`. This automatically covers every transition/animation utility any downstream plan adds; no per-component guard needed.

    Ambient glow CSS rule (FOUND-05, D-04/D-05 — static, GPU-cheap, no animation): add a `.ambient-glow` rule: `position: fixed; inset: 0; z-index: -1; pointer-events: none;` with `background: radial-gradient(ellipse 60% 50% at 100% 0%, color-mix(in oklch, var(--gradient-teal) 14%, transparent), transparent 70%);`. One corner glow only, referencing the shared `--gradient-teal` token so it stays in sync with the criticality gradient. Do NOT add `filter: blur()` (site-header already has one backdrop-blur; a second blur layer is a banned performance cost). Because it never animates, it needs no reduced-motion entry.

    Create `src/lib/motion.ts` (new) matching the `src/lib/utils.ts` shape — named string-constant exports, no default export, no logic: `transitionMicro = "transition-colors duration-micro ease-out"`, `transitionBase = "transition-all duration-base ease-out-soft"`, `pressFeedback = "active:translate-y-px"`. These are the shared className constants that Plan 02's tab components consume.
